Task 1: Symbolic, Unconditioned Generation

Dataset: EMOPIA_1.0. Output file: symbolic_unconditioned.mid.

1. Exploratory Analysis and Preprocessing

EMOPIA is a pop-piano dataset organized around perceived emotion. The local copy contains transcribed MIDI clips, YouTube metadata, raw tagging lists, and a label.csv file with one of four valence-arousal quadrants for each clip. This task ignores the emotion labels during generation because it is unconditioned, but the labels are still useful for checking coverage and making a song-level split.

I parsed the MIDI files directly, extracted note-on/note-off pairs, quantized time to sixteenth-note steps, and represented each clip as an event sequence: TS_k advances time by k sixteenth-note steps and N_pitch_duration_velocity places a note at the current time. Durations are capped at 32 steps and velocities are bucketed into 7 nonzero bins. Splits are made by YouTube song ID so clips from the same source song do not appear in both training and test sets.

After generation, I also applied a small sound-design post-processing step to make the MIDI less percussive: generated note velocities are softened, very short notes are given a longer minimum duration, the tempo is lowered, and the instrument is changed from piano to a string ensemble. These changes affect the rendered sound but not the symbolic n-gram training objective.

2. Modeling

The ML problem is next-event modeling: estimate p(x_t | x_{t-1}, ..., x_{t-n+1}) over symbolic MIDI events, then sample events autoregressively. I trained a variable-order n-gram model with maximum order 2. When an exact context is rare or unseen, the model interpolates with shorter contexts, ending in a smoothed unigram model. This is more expressive than a first-order Markov chain because it can preserve short timing and note-pattern motifs, but it remains transparent and fast enough to train from scratch on the full dataset without GPUs or external packages.

Compared with an RNN or Transformer, this model has limited long-range form: it does not explicitly know about phrase boundaries, harmonic goals, or repetition over many measures. Its advantages are that it is robust on a small dataset, easy to audit, and directly optimizes a symbolic likelihood aligned with the generation process.

For the final sample, I increased the target length to make the generated piece longer and lowered the sampling temperature below 1.0. Lower temperature makes the model choose more likely next events, which usually produces safer, smoother, and less chaotic output.

3. Evaluation

For objective evaluation, I compare held-out next-token perplexity for unigram, bigram, trigram, and four-gram models. Lower perplexity means the model assigns higher probability to unseen EMOPIA clips. This does not prove musical quality: a model can have good perplexity but still wander harmonically. So I also report simple musical descriptors of the generated sample beside train/test averages.

4. Related Work

The EMOPIA paper introduced the dataset for emotion recognition and emotion-based music generation in pop piano. It pairs symbolic MIDI with emotion annotations in the four-quadrant valence-arousal space, which makes it natural for conditioned generation. This Task 1 model deliberately leaves the emotion labels out to estimate the overall symbolic distribution instead.

Prior symbolic music generation systems often use event-token language models: older systems use Markov or n-gram variants, while recent systems use recurrent networks or Transformer-like decoders. The same evaluation tension appears throughout this literature: likelihood-style metrics are useful for debugging, but final musical quality usually needs listening tests or human judgment.

References: Hung et al., EMOPIA: A Multi-Modal Pop Piano Dataset for Emotion Recognition and Emotion-based Music Generation; project page at annahung31.github.io/EMOPIA.
